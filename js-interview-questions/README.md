## How to Use

1. **Practice**: Open a question file and try to solve it yourself first
2. **Compare**: Check the answer file for multiple solution approaches
3. **Learn**: Each answer includes:
   - Multiple approaches (from basic to optimized)
   - Time and space complexity analysis
   - Test cases and examples

## Running Examples

```bash
# Run a specific question
node questions/01-palindrome.js

# Run the answer with all approaches
node answers/01-palindrome.js
```

## Key Concepts Covered

- **Time Complexity**: Big O analysis for each solution
- **Space Complexity**: Memory usage considerations
- **Multiple Approaches**: From brute force to optimized solutions
- **Common Patterns**:
  - Two Pointers
  - Sliding Window
  - Hash Maps/Sets
  - Recursion & Memoization
  - Dynamic Programming
  - Stack/Queue usage

## Tips for Interviews

1. Always clarify requirements before coding
2. Start with brute force, then optimize
3. Discuss time/space complexity
4. Test with edge cases
5. Write clean, readable code
